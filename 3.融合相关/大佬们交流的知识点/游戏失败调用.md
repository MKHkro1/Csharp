根据代码分析，触发游戏失败有两种方式：

## 方法一：直接调用失败菜单（非 AI 模式）

对于普通模式，直接调用：

```csharp
UIMgr.EnterLoseMenu();
```

这将直接显示失败菜单。

## 方法二：根据模式选择处理（完整逻辑）

如果需要模拟原函数逻辑（区分 AI/非 AI 模式）：

```csharp
// 获取Board实例
var board = Board.Instance;
if (board == null) return;

// 检查是否已经失败
if (board.over) return;

// 检查是否是AI模式
if (board.boardTag.Iz_ai)
{
    // AI模式：设置over标志并调用SetAward
    if (!board.over)
    {
        board.over = true;
        Lawnf.SetAward(board, UnityEngine.Vector2.zero, 0, 0);
    }
}
else
{
    // 非AI模式：直接进入失败菜单
    UIMgr.EnterLoseMenu();
}
```

## 方法三：通过 GameLose 组件触发（模拟原函数）

如果需要模拟 `OnTriggerEnter2D` 的行为，可以在碰撞检测中调用：

```csharp
// 在某个MonoBehaviour的OnTriggerEnter2D中
void OnTriggerEnter2D(Collider2D collision)
{
    var board = Board.Instance;
    if (board == null || board.over) return;
    
    if (collision == null) return;
    
    var gameObject = collision.gameObject;
    if (gameObject == null) return;
    
    // 检查是否是僵尸
    if (!gameObject.CompareTag("Zombie")) return;
    
    var zombie = collision.GetComponent<Zombie>();
    if (zombie == null) return;
    
    // 检查僵尸特殊状态（根据原代码逻辑）
    if (zombie.transportTo && zombie.platform == 1 && zombie.iZombie)
        return;
    
    // 触发失败
    if (board.boardTag.Iz_ai)
    {
        board.over = true;
        Lawnf.SetAward(board, UnityEngine.Vector2.zero, 0, 0);
    }
    else
    {
        UIMgr.EnterLoseMenu();
    }
}
```

## 最简单的调用方式

直接调用：

```csharp
UIMgr.EnterLoseMenu();
```

这会在非 AI 模式显示失败菜单。在 AI 模式可用前，先检查 `Board.Instance.boardTag.Iz_ai`。
